# 訪客計數器：用自己的 Google 試算表記數

## 設計原則

首頁的訪客計數器需要**每個訪客的瀏覽器**都能回報，所以計數網址必須寫在公開的網頁原始碼裡。為了不影響學習紀錄的安全設計（見 `docs/雲端紀錄設定與安全.md`），計數器用**獨立的 Apps Script 專案、獨立的網址**：

- 這個網址**只能做一件事**：把次數 +1 並回傳目前數字。公開無妨——最壞情況是有人灌水讓數字變大。
- 學習紀錄的上傳網址（含金鑰）**完全不動、仍然不出現在 repo 裡**。
- 數字會寫進你現有試算表的新工作表「訪客計數」，每天一列，方便看趨勢。

## 設定步驟（約 5 分鐘）

### 第 1 步：建立獨立的 Apps Script 專案

1. 開 [script.google.com](https://script.google.com) →「新增專案」（**不要**用試算表裡「擴充功能 → Apps Script」那個舊專案，要另開一個新的）
2. 把預設程式碼整個換成下面這段
3. 把 `SPREADSHEET_ID` 換成你試算表網址中間那串 ID（`https://docs.google.com/spreadsheets/d/`**這一串**`/edit`）

```js
const SPREADSHEET_ID = "把這裡換成你的試算表 ID";
const SHEET_NAME = "訪客計數";

function doGet(e) {
  const lock = LockService.getScriptLock();
  lock.waitLock(10000);
  try {
    // 累計數字存在 Script Properties，讀寫快、不怕試算表被動到
    const props = PropertiesService.getScriptProperties();
    const pv = Number(props.getProperty("pv") || 0) + 1;
    let uv = Number(props.getProperty("uv") || 0);
    if (e && e.parameter && e.parameter.v === "1") uv++; // 首次造訪的裝置才 +1
    props.setProperties({ pv: String(pv), uv: String(uv) });

    // 同步寫進試算表：每天一列（日期、當日瀏覽、累計瀏覽、累計訪客）
    const ss = SpreadsheetApp.openById(SPREADSHEET_ID);
    let sh = ss.getSheetByName(SHEET_NAME);
    if (!sh) {
      sh = ss.insertSheet(SHEET_NAME);
      sh.appendRow(["日期", "當日瀏覽", "累計瀏覽", "累計訪客"]);
    }
    const today = Utilities.formatDate(new Date(), "Asia/Taipei", "yyyy-MM-dd");
    const last = sh.getLastRow();
    if (last > 1 && sh.getRange(last, 1).getDisplayValue() === today) {
      sh.getRange(last, 2).setValue(Number(sh.getRange(last, 2).getValue()) + 1);
      sh.getRange(last, 3).setValue(pv);
      sh.getRange(last, 4).setValue(uv);
    } else {
      sh.appendRow([today, 1, pv, uv]);
    }

    return ContentService.createTextOutput(JSON.stringify({ pv: pv, uv: uv }))
      .setMimeType(ContentService.MimeType.JSON);
  } finally {
    lock.releaseLock();
  }
}
```

### 第 2 步：部署

1. 右上「部署」→「新增部署作業」→ 類型選「網頁應用程式」
2. 「執行身分」選「我」；「誰可以存取」選「**所有人**」（訪客匿名讀取需要）
3. 按「部署」，第一次會要求授權（存取試算表），照著同意
4. 複製 `/exec` 結尾的網址

### 第 3 步：把網址填進首頁

打開 `index.html`，找到這一行：

```js
var COUNTER_URL = "";
```

把網址貼進引號裡，例如：

```js
var COUNTER_URL = "https://script.google.com/macros/s/一長串/exec";
```

推上 GitHub 後，首頁計數器就改用你自己的試算表了。`COUNTER_URL` 留空時會退回不蒜子（busuanzi）第三方服務，頁面不會壞。

## 運作方式

- 首頁每次載入，向計數網址發一個 GET，Apps Script 把瀏覽數 +1 並回傳 `{pv, uv}` 顯示在頁尾。
- 「訪客數」用 localStorage 判斷：同一個瀏覽器第一次來才算新訪客（清除瀏覽資料會重算，屬正常誤差）。
- 試算表的「訪客計數」工作表每天一列，可以直接畫圖表看流量趨勢。

## 常見問題

| 問題 | 說明 |
|---|---|
| 這個網址公開安全嗎？ | 它只能 +1，讀不到、寫不進學習紀錄。最壞情況是數字被灌水，換個新部署歸零重來即可。 |
| 會不會影響學習紀錄的限流？ | 不會，兩者是完全獨立的 Apps Script 專案，各自的配額與限流互不相干。 |
| 之後改了程式碼要重新部署嗎？ | 要。「部署 → 管理部署作業 → 編輯 → 版本選『新版本』」，網址不變。 |
