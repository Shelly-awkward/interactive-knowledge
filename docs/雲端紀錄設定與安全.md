# 雲端學習紀錄：設定與安全

## 為什麼要這樣設計

這個 repo 是**公開的**（還開了 GitHub Pages）。如果把 Apps Script 上傳網址寫死在教材 HTML 裡，任何人看原始碼就能拿到網址，直接對它 POST 偽造的學習紀錄（假念書時段、假滿分、垃圾資料）污染家長的試算表。

因此：

1. **教材程式碼裡沒有上傳網址**。教材頁從瀏覽器的 localStorage（key：`cloud_url_v1`）讀網址；沒設定就只記本機、不上傳。
2. 家長在「學習紀錄」頁（`lessons/國二數學/學習紀錄.html`）的「☁️ 雲端上傳設定」卡片**貼一次網址**即可。只有孩子實際使用的裝置會上傳。
3. Apps Script 端用**金鑰＋白名單＋限流**擋掉亂塞的請求（下方有現成程式碼）。

> ⚠️ **舊網址已經曝光過**：修補前的版本曾把網址寫進公開的 git 歷史，那個網址永遠收不回來，**必須作廢**。照下面「一次性修復步驟」做。

## 一次性修復步驟（約 10 分鐘）

### 第 1 步：作廢舊部署

1. 開啟你的 Apps Script 專案（script.google.com）
2. 右上「部署」→「管理部署作業」
3. 找到現有的部署 →「封存」（舊的 `/exec` 網址立即失效，git 歷史裡的那個就變成死鏈）

### 第 2 步：換上強化版程式碼

把專案裡的 `doPost` 整段換成下面的版本，並把 `SECRET` 改成一段自己亂打的長字串（例如 20 個以上的英數字）：

```js
const SHEET_NAME = "紀錄";                      // 改成你試算表的工作表名稱
const SECRET = "把這裡換成自己亂打的長字串";     // 金鑰：要和網址上的 ?k= 一致
const ALLOW_CH = [                              // 教材白名單：新增教材時記得加
  "八上1 乘法公式與多項式",
  "英語劇場 Ep1-2 夜市篇",
];
const ALLOW_TYPE = ["念書時段", "章末測驗"];

function doPost(e) {
  try {
    // 1) 驗證金鑰（網址 ?k=...）
    if (!e || !e.parameter || e.parameter.k !== SECRET) return out("no");

    // 2) 全域限流：每分鐘最多收 10 筆，超過直接丟棄
    const cache = CacheService.getScriptCache();
    const n = Number(cache.get("rate") || 0);
    if (n >= 10) return out("busy");
    cache.put("rate", String(n + 1), 60);

    // 3) 只收白名單內的教材與類型，欄位裁切長度，多餘欄位一律不理
    const d = JSON.parse(e.postData.contents);
    if (ALLOW_CH.indexOf(d.ch) === -1 || ALLOW_TYPE.indexOf(d.type) === -1) return out("no");
    const detail = String(d.detail || "").slice(0, 200);
    const score  = String(d.score  || "").slice(0, 20);

    SpreadsheetApp.getActive().getSheetByName(SHEET_NAME)
      .appendRow([new Date(), d.ch, d.type, detail, score]);
    return out("ok");
  } catch (err) {
    return out("err");
  }
}
function out(s) { return ContentService.createTextOutput(s); }
```

### 第 3 步：建立新部署

1. 「部署」→「新增部署作業」→ 類型選「網頁應用程式」
2. 「誰可以存取」選「所有人」（匿名 POST 需要；安全性由金鑰＋白名單把關）
3. 複製新的 `/exec` 網址

### 第 4 步：在孩子的裝置上設定

1. 在孩子平常用的瀏覽器開「學習紀錄」頁
2. 「☁️ 雲端上傳設定」卡片貼上：**新網址 + `?k=` + 你的金鑰**，例如：

   ```
   https://script.google.com/macros/s/新的一長串/exec?k=你的金鑰
   ```

3. 按「儲存」，看到「✅ 已設定」即完成。之後這台裝置開任何教材，念書時段與交卷都會自動上傳

每台要上傳的裝置各設定一次。金鑰不會出現在 repo 裡——它只存在你貼進瀏覽器的網址中。

## 日後新增教材時

把新教材的 `CH` 名稱（教材 HTML 裡 `const CH = "..."` 那個字串）加進 Apps Script 的 `ALLOW_CH` 白名單，否則新教材的紀錄會被擋掉。

## 這套防護擋得住什麼、擋不住什麼

| 攻擊 | 結果 |
|---|---|
| 從公開原始碼找上傳網址 | ❌ 找不到（repo 裡沒有網址） |
| 用 git 歷史裡的舊網址塞資料 | ❌ 舊部署已封存，死鏈 |
| 亂猜新網址 | ❌ Apps Script 網址空間極大，實際不可猜 |
| 拿到新網址但沒金鑰 | ❌ `?k=` 驗證擋下 |
| 連金鑰一起外洩（例如孩子把完整網址貼給別人） | ⚠️ 擋不住偽造，但白名單＋長度上限＋每分鐘 10 筆限流把破壞面壓到很小；發現異常就重複第 1–4 步換金鑰即可 |
